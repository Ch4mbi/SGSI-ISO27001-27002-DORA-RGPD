# **3.Términos y definiciones** 

| Término | Definición |
| ----- | ----- |
|  **SGSI**  | Sistema de gestión de seguridad de la información, consta de un conjunto de principios o procedimientos que se usan para identificar riesgos y definir pasos de mitigación de los mismos garantizando así que las empresas puedan mantener seguros los datos y la información, para proteger los sistemas de banca online, como bases de datos, apps, … asegurando así la confidencialidad, integridad y disponibilidad de la información frente a amenazas. (AG, 2025) |
|  **Activos (Financieros)**  | Cualquier dato o conjunto de información o sistema de una organización o entidad que es valioso para la misma. En ChamBank los activos más críticos la base de datos de los clientes, los sistemas internos, la infraestructura cloud donde se almacenan las transacciones, …   (Santander, 2024) |
|  **Amenazas**  | Probabilidad de que pase un incidente que pueda causar daños de algún tipo a la organización, ya sea parar servicios, robar información, escalar privilegios, ... o usar otra clase de técnicas como malware, phishing, ransomware, etc. (Kelly, 2021) |
|  **Vulnerabilidad**  | Fallo de un sistema que un atacante puede aprovechar para obtener beneficios de diferentes tipos (Accesos como administrador, permisos, conexiones, robo de datos, etc). En ChamBank, algunas vulnerabilidades son: errores de programación, malas configuraciones, falta de MFA, … En ChamBank, es posible encontrarse con dichas vulnerabilidades en las apps web del banco, sistemas sin cifrado, o incluso los errores humanos se consideran vulnerabilidades. |
|  **Riesgos (Bancarios)**  | Probabilidad de que una vulnerabilidad sea explotada o que una amenaza se “haga realidad” y que afecte negativamente a una empresa. Es decir, que, si un atacante logra usar una vulnerabilidad para acceder al sistema, puede llevar a cabo varias acciones maliciosas. (Wikipedia, 2020) |
|  **Controles de seguridad**  | Son parámetros implementados para proteger diversos aspectos de una. Se implementan para evitar, detectar y/o minimizar los riesgos de seguridad. Algunas medidas preventivas del banco serían: MFA, cifrar datos enviados, firewalls bien configurados, … Estos controles se pueden aplicar en diversas capas: física, técnica, o controles organizativos. (IBM, 2025) |
|  **CIA/CID**  | Garantiza que la información solo sea accesible para usuarios autorizados, que los datos se mantengan exactos sin modificaciones no autorizadas y  que los sistemas y servicios estén operativos y accesibles cuando se necesiten. |
|  **Incidentes de seguridad**  | Evento/os que pueden afectar al CID de una compañía, perjudicándola. Dichos eventos, cabe destacar, que violan las políticas de las organizaciones. Estos incidentes pueden ser accesos no autorizados, caídas del sistema, logs sospechosos a las apps del banco o a cuentas, … En ChamBank, los incidentes de seguridad pueden ser: accesos fraudulentos, filtraciones de datos, fraudes, … Los incidentes deben de ser detectados, analizados y mitigados con planes de respuesta ante incidentes.  (Santander, 2024) |
|  **Superficie de ataque**  | Es el número de todos los posibles puntos por los que un atacante puede intentar acceder ilegalmente a una red de una compañía, por ejemplo. Cuantos más servicios vinculados tiene la entidad, más superficie de ataque existe, por ejemplo, si una compañía se pasa a la nube, la superficie aumentará y habrá que defender más superficie.  (Fortinet, 2026) |
|  **Riesgo residual**  | Es el riesgo que permanece después de haber implementado muchas medidas de seguridad en las que se haya pensado. En Chambank está formado por posibles vulnerabilidades zero-day, o errores humanos, o similares. (Anon., 2023) |
